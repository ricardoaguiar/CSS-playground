The border class:
This turquoise with a red border box is a single html div element with a single css class attached to it.
On the css we created two classes, border and background with a single font property. The font property applies to both classes. .border,
.background {
  font-weight: 800;
} The background class is also declared alongside the border class and has the same font-weight property but it has no effect on this element because we are already calling the border class and it is responsible to handle the font-weight on this instance: <div class="border"></div> How come we have a green background and a red border then? Where does it come from? That's when the CSS hierarchy comes into play. These color and other properties we declare them on another .border class. That means we can use the font-weight in two classes and leave the other border properties on its own class: .border {
  width: auto;
  height: auto;
  background: white;
  padding: 10px;
  border: 1px solid red;
  background: turquoise;
}
